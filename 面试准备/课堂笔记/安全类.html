是一个很重要的知识面，但是不很难 CSRF：通常称为跨站请求伪造，英文名 Cross-site request forgery 攻击原理：根据你网站的漏洞，去帮你自动执行那些接口，并且依赖于用户登陆过网站 用户登录网站A，网站A核查这个身份是不是正确，如果正确就下发cookie，cookie保存在用户浏览器中。这就完成了一次身份认证的过程。 用户又访问了网站B，网站B中有一个引诱用户点击的链接，链接指向网站A的api的接口尤其是这个网站又是一个get的接口。访问到网站A后浏览器上传cookie，进行身份重新确认，发现是合法用户，就执行了这个接口的动作。（新浪微博莫名多了很多粉丝很多关注） CSRF能攻击的两大因素： 1、网站的某个接口中存在这种漏洞 2、用户的确在哪个网站登录过 目前防御 CSRF 攻击主要有三种策略：验证 HTTP Referer 字段；在请求地址中添加 token 并验证；在 HTTP 头中自定义属性并验证。 防御： Token验证 Referer验证 隐藏令牌 XSS cross-site scripting 跨站脚本攻击 攻击原理：向你的页面注入脚本，评论区是最好的位置。在脚本中写入他想做的事。 防御措施：想方设法让他的脚本不执行
